🚀 My CI/CD Pipeline Process

This page outlines the steps I followed to set up a CI/CD pipeline using AWS CodePipeline and S3.

📦 Step 1: Create an S3 Bucket

I created an S3 bucket named aws-cicd-static-website and enabled static website hosting.

🐙 Step 2: Set Up GitHub Repository

I created a GitHub repository and added my static website files (index.html, style.css).

⚙️ Step 3: Create AWS CodePipeline

I set up a CI/CD pipeline using AWS CodePipeline. The pipeline pulls code from GitHub and deploys it to the S3 bucket.

🧪 Step 4: Test the Pipeline

I pushed changes to the GitHub repository and verified that the pipeline automatically deployed the updates to the S3 bucket.

🌐 Step 5: View the Website

I accessed the website using the S3 bucket endpoint: http://aws-cicd-static-website.s3-website-us-east-1.amazonaws.com/.

🚀 Next Steps

In the future, I plan to:

  • 🔗 Add a custom domain using Route 53.
